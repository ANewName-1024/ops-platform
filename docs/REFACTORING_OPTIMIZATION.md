# 重构方案优化建议

**审核日期：2026-03-19**
**审核人：小爪**

---

## 1. 服务拆分优化

### 当前方案
```
workflow-service (8081)
knowledge-service (8082)  
chat-service (8083)
```

### 优化建议

| 问题 | 建议 | 理由 |
|------|------|------|
| knowledge + AI 耦合 | 拆分为 knowledge-service + ai-service | AI 能力可复用 |
| notification 缺失 | 添加 notification-service | 通知独立部署 |
| admin 过于简单 | 并入 gateway 或独立 | 根据实际复杂度 |

### 优化后

```
workflow-service (8081)    # 工作流
knowledge-service (8082)  # 知识库存储
ai-service (8083)         # AI 能力 (可复用)
chat-service (8084)       # 对话
notification-service (8085) # 通知
gateway (8080)            # 网关 + admin
```

## 2. 数据层优化

### 当前问题
- 单一数据库
- 无读写分离
- 无分库分表

### 优化建议

| 方案 | 描述 | 复杂度 |
|------|------|--------|
| 读写分离 | 主从 PostgreSQL | 中 |
| 多租户 | 按 tenant_id 分表 | 高 |
| 分库 | 按业务拆分 | 中 |

### 推荐：渐进式

```
Phase 1: 单一数据库 + 分表
Phase 2: 读写分离
Phase 3: 按服务分库
```

## 3. 安全设计增强

### 当前缺失

| 缺失项 | 建议 |
|--------|------|
| API 签名验签 | 添加请求签名机制 |
| 敏感数据加密 | 字段级加密 |
| 审计日志 | 完整操作日志 |
| 限流熔断 | Spring Cloud CircuitBreaker |
| 黑名单/IP 白名单 | 基础防护 |

### 优化架构

```
Client → WAF → Gateway → Auth → Service
                ↓
          Rate Limiter
          Circuit Breaker
          Audit Log
```

## 4. 可观测性增强

### 当前缺失

| 组件 | 建议 |
|------|------|
| 日志 | ELK (已有) → 升级为 Loki |
| 链路 | SkyWalking (比 Zipkin 更完善) |
| 告警 | AlertManager + Prometheus |
| 通知 | 集成到现有告警系统 |

### 优化后监控栈

```
Metrics: Prometheus + Grafana
Logs: Loki + Grafana
Traces: SkyWalking
Alerts: AlertManager
```

## 6. 成本优化

### 当前问题

| 问题 | 影响 | 优化建议 |
|------|------|----------|
| 全量 AI 调用 | 成本高 | 缓存 + 降级策略 |
| 大文件存储 | 存储贵 | 接入对象存储 (MinIO/OSS) |
| 实时日志 | 带宽高 | 采样 + 异步写入 |

### 推荐策略

```
AI 成本优化:
├── 本地模型优先 ( Ollama )
├── 缓存相似问题答案
├── 限流 + 配额控制
└── 按需调用高级模型

存储成本优化:
├── 冷热分离
├── 文档 -> 对象存储
└── 定期清理过期数据
```
